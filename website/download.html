Get AISplinter

Self-hosted in minutes. Choose the path that fits your stack — Docker, standalone, or embedded in Next.js.

1. Clone the repo

git clone https://github.com/javorvatchkov/aisplinter.git cd aisplinter

2. Configure environment

cp packages/server/.env.example apps/server/.env # Edit apps/server/.env — set DATABASE_URL and AISPLINTER_OPENROUTER_KEY

3. Start with Docker Compose

docker compose up --build

The setup wizard runs on http://localhost:8747 the first time the database is empty.

1. Requirements

  • Node.js 20+ and pnpm 9+
  • PostgreSQL 14+ (local or Neon)
  • An OpenRouter management key

2. Install & configure

git clone https://github.com/javorvatchkov/aisplinter.git cd aisplinter pnpm install cp packages/server/.env.example apps/server/.env # Edit apps/server/.env

3. Run the dev server

pnpm dev

Turborepo starts the standalone server on port 8747.

1. Install the server package

pnpm add @aisplinter/server

2. Create the singleton

import { createAisplinterApp } from '@aisplinter/server'; import { createAisplinterSingleton } from '@aisplinter/server/next'; export const getAisplinterApp = createAisplinterSingleton(async () => { const { app } = await createAisplinterApp({ basePath: '/api/aisplinter' }); return app; });

3. Wire the route

// app/api/aisplinter/[...path]/route.ts import { getAisplinterApp } from '@/lib/aisplinter-server'; import { createNextRouteHandlers } from '@aisplinter/server/next'; export const { GET, POST, PUT, PATCH, DELETE, OPTIONS } = createNextRouteHandlers(getAisplinterApp);

The admin panel is available at /api/aisplinter/admin after setup.

Quick reference

System requirements

◆

Runtime

Node.js 20+ with native fetch. PNPM 9+ for workspace management.

◆

Database

PostgreSQL 14+. Works with local, Docker, or managed (Neon, Supabase).

◆

Upstream

OpenRouter management key or LiteLLM proxy for provider-neutral access.

◆

Host

Any host that can call app.fetch() — Hono, Express, Next.js, Bun, Deno.
